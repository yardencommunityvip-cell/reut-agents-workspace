# Learning Log & Style Preferences

## What Works (Successes)
- Posts that end with personal questions about "Team A vs Team B" get 40% more comments.
- Using specific "Heblish" terms (e.g., "אייקוני", "רד קרפט") feels more authentic to the members.
- Summaries of "The Drama So Far" are highly appreciated by busy members.

## What to Avoid (Mistakes)
- Do not use overly formal Hebrew (e.g., avoid "ברם" or "אנוכי").
- Do not translate celebrity names into Hebrew if they are better known in English.
- Avoid "clickbait" titles that don't deliver—keep it classy and expert.

## Style Corrections Log — [2026-03-25] v2 Rewrite

**Mistakes Made in Previous Draft (v1 — Zendaya/Tom Holland post):**

1. DASH LISTS — Used bullet points with dashes throughout the post body. This makes the output look like a bot-generated news summary, not a message from a community leader. Never use "• " or "- " inside the post body. Use natural paragraph transitions instead.

2. MID-TITLE HEADERS — Inserted bold headers like "מה קרה:", "הטוויסט הקולטורלי:", "בינתיים:" inside the post. This fragments the flow and kills the personal voice. One unified text block only.

3. CORPORATE-PLAYFUL EMOJI ABUSE — Opened each paragraph or bullet with an emoji (🎬✨, 📍, 🧠, 🍿, 💬). This creates the "AI content" visual pattern. Emojis are allowed only once or twice total, purely for vibe/tone, never as structural markers.

4. METADATA FOOTER — Ended the post with "נכתב על ידי YMD Pop Culture Curator" and a sources list. Posts should feel like a personal message from Yarden, not a published article. Remove all attribution/source lines from inside the final post.

5. STALE STORY — Used the Zendaya/Law Roach secret wedding angle, which had already been circulating for weeks. Always verify the story is from the last 12 hours and hasn't been discussed to death. Cross-check freshness across at least two sources before committing to a topic.

**Rules Confirmed and Active:**
- Post = one unbroken flowing narrative: hook → tea → cultural insight → human connection → closer
- Cultural analysis must be woven into the prose, not sectioned off
- Conversation starter at the end must feel genuine and sophisticated, not like a "marketing question"
- Voice: Yarden's bestie tone — sharp, warm, Heblish, never formal, never robotic

## Style Corrections Log — [2026-03-25] v3 Rewrite (URGENT)

**Mistakes Made in Previous Draft (v2 — Rihanna post):**

1. LONG DASHES (—) — FORBIDDEN. The em-dash is a hard banned character inside post body. It creates a robotic, editorial tone. Replace with a comma, a period, or restructure the sentence entirely. No exceptions.

2. DRAMATIC OPENING PHRASES — FORBIDDEN. Phrases like "יש משהו שקרה היום שאני צריכה לדבר איתכן עליו" or "מה שמשתק אותי" sound like a news anchor performing emotion, not a bestie sharing a real take. Use grounded, analytical hooks instead. Approved alternatives: "שימו לב לסיפור הזה", "מה שמעניין בסיפור הזה הוא...", "שימו לב לקונטרסט הספציפי הזה".

3. TRANSLATION ERRORS / NON-HEBLISH VOCAB — FORBIDDEN. "עקיבה" for stalking is overly formal Hebrew. Use "סטוקינג". "רשתות חברתיות" sounds like a TV news report. Use "סושיאל". Always default to the Heblish/phonetic version of any term that has a common English-origin equivalent in Israeli slang.

**Vocabulary Reference (Heblish Defaults):**
- Stalking = סטוקינג (not עקיבה)
- Social media = סושיאל (not רשתות חברתיות)
- Parasocial = פארה-סושיאל
- Shooting = שיטינג
- Casting = קאסטינג
- Vibe = וייב
- Content = קונטנט

**Full Forbidden List (cumulative):**
- Long dashes (—) inside post body
- Bullet points or dash lists inside post body
- Mid-title headers inside post body
- Dramatic first-person phrases ("אני צריכה לדבר איתכן", "מה שמשתק אותי")
- Metadata footer (sources, written by, date stamps)
- Emoji used as structural/paragraph markers
- Stale stories (older than 12 hours)
- Overly formal Hebrew vocabulary when Heblish equivalent exists
